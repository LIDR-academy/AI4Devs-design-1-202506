# LTI - Applicant Tracking System

## 1. Descripción del Software LTI

### Valor Añadido y Ventajas Competitivas

**LTI (Lead Talent Intelligence)** es un ATS (Applicant-Tracking System) de nueva generación que revoluciona la gestión de talento mediante la integración de inteligencia artificial avanzada, automatización inteligente y colaboración en tiempo real.

#### Ventajas Competitivas Clave:

1. **IA Generativa Avanzada**: Utiliza modelos de lenguaje para generar descripciones de trabajo, resúmenes de candidatos, preguntas de entrevista personalizadas y feedback automático.

2. **Matching Inteligente**: Sistema de emparejamiento que analiza CVs, evaluaciones y pruebas técnicas para identificar candidatos ideales con precisión superior al 90%.

3. **Colaboración en Tiempo Real**: Herramientas de comunicación integradas que permiten a reclutadores, managers y equipos coordinar decisiones de contratación instantáneamente.

4. **Automatización End-to-End**: Flujos de trabajo automatizados que reducen el tiempo de contratación en un 60%, desde la publicación de ofertas hasta la incorporación.

5. **Analytics Predictivos**: Dashboards inteligentes que predicen el tiempo de contratación, éxito de candidatos y optimización de pipelines.

6. **Integración Universal**: APIs abiertas que se conectan con LinkedIn, GitHub, sistemas de evaluación técnica, calendarios y herramientas de HR existentes.

## 4. Casos de Uso Principales

### 4.1 Caso de Uso 1: Publicar Oferta y Recibir Aplicaciones

**Actor Principal**: Reclutador

**Precondiciones**: 
- El reclutador tiene cuenta activa en LTI
- Existe un puesto de trabajo definido

**Flujo Principal**:
1. El reclutador accede a LTI y selecciona "Crear nueva oferta"
2. El sistema solicita información básica del puesto (título, descripción, requisitos, ubicación, salario)
3. El reclutador proporciona información mínima o carga un documento
4. El sistema utiliza IA para generar una descripción optimizada y sugerencias de mejoras
5. El reclutador revisa y aprueba la descripción generada
6. El reclutador configura canales de publicación (LinkedIn, Indeed, página web, etc.)
7. El sistema publica automáticamente la oferta en los canales seleccionados
8. Los candidatos comienzan a aplicar a través de los canales
9. El sistema recibe y almacena todas las aplicaciones en un pipeline único
10. El sistema notifica al reclutador de nuevas aplicaciones

**Flujos Alternativos**:
- 4a. El reclutador rechaza la descripción generada y edita manualmente
- 6a. El reclutador decide publicar solo en algunos canales
- 8a. Un candidato aplica directamente desde la base de datos de talento de LTI

**Postcondiciones**:
- La oferta está publicada en los canales seleccionados
- Las aplicaciones se están recibiendo y almacenando
- El pipeline está activo para el puesto

### 4.2 Caso de Uso 2: Evaluar y Clasificar Candidatos con IA

**Actor Principal**: Reclutador

**Precondiciones**:
- Existen aplicaciones recibidas para un puesto
- El sistema tiene acceso a APIs de evaluación

**Flujo Principal**:
1. El reclutador accede al pipeline del puesto
2. El sistema muestra lista de candidatos ordenados por scoring inicial
3. El reclutador selecciona un candidato para evaluación detallada
4. El sistema analiza el CV del candidato usando IA
5. El sistema compara el perfil con los requisitos del puesto
6. El sistema genera un score de matching (0-100)
7. El sistema sugiere evaluaciones técnicas o pruebas adicionales
8. Si el candidato pasa el screening inicial, el sistema envía automáticamente pruebas de evaluación
9. El candidato completa las evaluaciones
10. El sistema procesa los resultados y actualiza el score
11. El sistema actualiza el ranking del candidato en el pipeline
12. El reclutador recibe notificación con el resultado completo

**Flujos Alternativos**:
- 4a. El CV es de baja calidad o ilegible → El sistema marca para revisión manual
- 7a. El candidato no cumple requisitos mínimos → El sistema marca como "No apto" automáticamente
- 9a. El candidato no completa las evaluaciones en tiempo → El sistema envía recordatorio

**Postcondiciones**:
- Los candidatos están evaluados y clasificados
- El pipeline muestra rankings actualizados
- Se han identificado candidatos calificados para siguiente fase

### 4.3 Caso de Uso 3: Colaboración y Aprobación de Contratación

**Actor Principal**: Reclutador, Hiring Manager

**Precondiciones**:
- Existe un candidato calificado en etapa final
- Se han completado entrevistas técnicas y culturales

**Flujo Principal**:
1. El reclutador marca un candidato como "Listo para decisión final"
2. El sistema notifica al Hiring Manager asignado
3. El Hiring Manager accede al perfil completo del candidato
4. El sistema muestra resumen ejecutivo, evaluaciones, entrevistas y feedback
5. El Hiring Manager puede ver comentarios previos del equipo
6. El Hiring Manager agrega su feedback y comentarios
7. El Hiring Manager toma decisión: "Aprobar", "Rechazar" o "Solicitar más información"
8. Si aprueba, el sistema notifica al reclutador
9. El reclutador genera oferta de trabajo con IA
10. El sistema envía la oferta al candidato
11. El candidato responde (acepta/rechaza/negocia)
12. Si acepta, el sistema inicia flujo de onboarding automático
13. Todos los stakeholders reciben notificación de la contratación

**Flujos Alternativos**:
- 7a. El Hiring Manager solicita más información → El sistema notifica al reclutador para seguimiento
- 7b. El Hiring Manager rechaza → El sistema marca candidato como "Rechazado" y notifica
- 11a. El candidato negocia términos → El sistema permite contraoferta y re-aprobación

**Postcondiciones**:
- Se ha tomado decisión sobre el candidato
- Si aprobado, la oferta está enviada o el candidato está en proceso de onboarding
- Todos los stakeholders están informados

## 6. Diseño del Sistema a Alto Nivel

### 6.1 Arquitectura General

LTI sigue una arquitectura de microservicios moderna basada en cloud, diseñada para escalabilidad, alta disponibilidad y fácil integración.

#### Componentes Principales:

1. **API Gateway**: Punto de entrada único que maneja autenticación, rate limiting y enrutamiento
2. **Servicios de Dominio**:
   - **User Service**: Gestión de usuarios, autenticación y autorización
   - **Job Service**: Gestión de puestos de trabajo y publicaciones
   - **Candidate Service**: Gestión de candidatos y aplicaciones
   - **Assessment Service**: Integración con proveedores de evaluaciones
   - **Interview Service**: Programación y gestión de entrevistas
   - **Matching Service**: Motor de matching con IA
   - **Notification Service**: Envío de emails, SMS y push notifications
   - **Analytics Service**: Procesamiento de métricas y reportes
3. **Servicios de IA**:
   - **AI Orchestrator**: Coordina llamadas a diferentes modelos de IA
   - **Content Generation**: Generación de descripciones, ofertas, etc.
   - **CV Analysis**: Análisis y extracción de información de CVs

### 7.4 Código (Nivel 4) - Pseudocódigo del Matching Algorithm

```python
class MatchingEngine:
    def calculate_match_score(self, candidate: Candidate, job: Job) -> float:
        """
        Calcula el score de matching entre un candidato y un puesto
        Retorna un valor entre 0 y 100
        """
        # 1. Extraer información estructurada
        candidate_profile = self.cv_analyzer.extract(candidate.resume)
        job_requirements = self.requirement_parser.parse(job)
        
        # 2. Generar embeddings
        candidate_embedding = self.embedding_generator.generate(
            candidate_profile.to_text()
        )
        job_embedding = self.embedding_generator.generate(
            job_requirements.to_text()
        )
        
        # 3. Calcular similitud semántica
        semantic_score = self.similarity_calculator.cosine_similarity(
            candidate_embedding,
            job_embedding
        ) * 100  # Normalizar a 0-100
        
        # 4. Calcular scores específicos
        skill_match = self.calculate_skill_overlap(
            candidate_profile.skills,
            job_requirements.required_skills
        )
        
        experience_score = self.calculate_experience_match(
            candidate_profile.years_experience,
            job_requirements.min_experience
        )
        
        education_score = self.calculate_education_match(
            candidate_profile.education,
            job_requirements.education_requirements
        )
        
        location_score = self.calculate_location_match(
            candidate_profile.location,
            job_requirements.location,
            job_requirements.remote_allowed
        )
        
        # 5. Scoring ponderado
        final_score = (
            semantic_score * 0.40 +      # 40% peso semántico
            skill_match * 0.30 +         # 30% skills
            experience_score * 0.15 +    # 15% experiencia
            education_score * 0.10 +     # 10% educación
            location_score * 0.05        # 5% ubicación
        )
        
        return round(final_score, 2)
    
    def calculate_skill_overlap(self, candidate_skills, required_skills):
        """Calcula el porcentaje de overlap de skills"""
        if not required_skills:
            return 100.0
        
        matched = len(set(candidate_skills) & set(required_skills))
        total_required = len(required_skills)
        
        return (matched / total_required) * 100
```
