## 2️⃣ Estructura de Carpetas (Monorepo)

> **Nota:** Describir y dibujar la estructura en el markdown. No es necesario crear archivos reales ahora.

### 📁 Estructura Principal

```
/apps
  /web                   # Next.js Frontend
    /src
      /app               # App Router de Next.js
        /jobs            # Contexto Jobs integrado con ruta Next.js
        /applications    # Contexto Applications integrado con ruta Next.js
        /dashboard       # Contexto Dashboard integrado con ruta Next.js
        /screening       # Contexto Screening (futuro)
        /interviews      # Contexto Interviews (futuro)
        /offers-hiring   # Contexto Offers-Hiring (futuro)
        /people          # Contexto People (futuro)
        /notifications   # Contexto Notifications (futuro)
        /analytics       # Contexto Analytics (futuro)
  /api                   # NestJS Backend
  
/packages
  /shared-kernel         # Types comunes, Result/Either, errors
  /ui                    # Design system
```

### 🏗️ Arquitectura DDD + Next.js App Router

**Innovación Arquitectónica:** Los bounded contexts se integran directamente dentro de las rutas de Next.js App Router, manteniendo tanto los principios DDD como las convenciones de Next.js.

Cada contexto en `src/app/[contexto]/` debe tener:

```
/src/app/[contexto]
  /page.tsx            # Página principal del contexto (Next.js)
  /domain              # Entities, value-objects, services, events, repo-ports
  /application         # Use-cases, DTOs, mappers, validators
  /infrastructure      # Prisma models, adapters, http, repositories
  /ui                  # Componentes React específicos del contexto
    /components        # Componentes reutilizables
    /hooks            # React hooks para integración con use-cases
```

### 🏗️ Arquitectura Implementada

**Patrón Revolucionario:** DDD Bounded Contexts integrados dentro de Next.js App Router

#### 🎯 Innovación Clave
En lugar de mantener los contextos DDD separados del framework (enfoque tradicional), esta implementación los integra directamente dentro de las rutas de Next.js App Router. Esto proporciona:

- **Cohesión:** Cada contexto es una ruta funcional de Next.js (`/jobs`, `/applications`, `/dashboard`)
- **Separación de Capas:** Mantiene la arquitectura DDD (Domain/Application/Infrastructure/UI) dentro de cada ruta
- **Escalabilidad:** Cada contexto puede crecer independientemente siguiendo su propia lógica de negocio
- **Mantenibilidad:** Los desarrolladores encuentran todo lo relacionado con un contexto en un solo lugar

#### 🔄 Patrones Implementados
- **CQRS:** Separación clara entre Commands y Queries en cada contexto
- **Repository Pattern:** Abstracciones para persistencia con implementaciones in-memory
- **Composite Repository:** Para agregación cross-context (Dashboard)
- **Dependency Injection:** Container centralizado para gestión de dependencias
- **React Integration:** Hooks especializados que conectan UI con Use Cases
